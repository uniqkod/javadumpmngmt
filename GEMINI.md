* prefer bash as terminal language
* project is defined in README.md file on the project root, read first
* all other documentation is in docs folder
  * docs/architecture.md contains the full architecture description
  * docs/requirements.md contains the full list of requirements
  * docs/design-decisions.md contains the design decisions made during the project
  * docs/backlog folder contains detailed major task definitions in sub type folder 
    * baseline requirement tasks are on the root of docs/backlog, will be coppied to docs/development folder when selected for implementation
    * bugs are in docs/backlog/bugs folder will be coppied to docs/development/fixes folder when selected for implementation
    * refactors are in docs/backlog/refactors folder will be coppied to docs/development/refactors folder when selected for implementation
    * features are in docs/backlog/features folder will be coppied to docs/development/features folder when selected for implementation
  * docs/development folder contains detailed major task definitions in sub type folder  
    * baseline requirement tasks are on the root of docs/development will be coppied to docs/commits when done
    * fixes are in docs/development/fixes folder will be coppied to docs/commits/fixes folder when done
    * refactors are in docs/development/refactors folder will be coppied to docs/commits/refactors folder when done
    * features are in docs/development/features folder will be coppied to docs/commits/features folder when done  
  * docs/commits folder contains detailed major commits definitions in sub type folder
    * ordinary commits are on the root of docs/commits
    * fixes are in docs/commits/fixes folder
    * refactors are in docs/commits/refactors folder
    * features are in docs/commits/features folder
  * docs/plan.md contains the full implementation plan
  * docs/project-structure-status.md contains the current status of the project structure and implemented features
  * docs/project-reevaluation-summary.md contains the summary of the project reevaluation after changing the architec
* all documentation files are in markdown format
